==「役割を超えたオーナーシップ」と「事業視点での技術選定」==を強みとしてきました。実装担当として参画したプロジェクトでも、画面仕様の曖昧さや組織横断の課題を見つけた際には、自ら仕様書を一から作成したり、Datadog のログ設計を見直したり、Claude Code のチーム共通基盤をプラグインとして整備したりと、「自分の役割の外」に落ちているボールを積極的に拾ってきました。結果として、自分一人ではなくチーム全体の生産性・品質改善につながる成果を生み出すことを意識しています。

セーフィーのリプレイス案件では、将来の技術スタック変更時にビジネスロジックを書き直さず済む構造を最初から狙って DI 設計を提案・採用しました。直近のアプリボットの案件でも、事業数値を分析チームと確認しながら、施策が事業KPIにどう寄与するかを意識しています。

変化への適応も大切にしています。React Native も Next.js も Claude Code も、参画時点では実務経験がありませんでしたが、調査・検証を重ねて成果につなげてきました。==「昨日の最善が、今日の最善とは限らない」==という前提で、自分のこだわりや過去のやり方をアップデートし続けることを心がけています。

今後はフロントエンドを主軸に置きつつ、現在着手中のバックエンド領域（Go）にも実装経験を広げ、プロダクト全体・組織全体への貢献の幅を広げていきたいと考えています。生成AIによる開発プロセスの変革が進む中で、技術選定とプロセス設計の両面から組織の生産性に貢献できるエンジニアを目指します。
